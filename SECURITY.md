# Security Policy

## Repository Scope

This is a research repository focused on agentic runtime architecture, orchestration safety, and reliability engineering.

This project does not provide support for bypassing service protections or security controls.

## Reporting a Vulnerability

Please report security issues privately.

Include:
- affected component/file
- reproduction steps or proof of concept
- expected impact
- suggested mitigation (if available)

Preferred channels:
- GitHub private security advisory (if enabled)
- Maintainer contact through a trusted private channel

Do not open public issues for unpatched vulnerabilities.

## Priority Security Areas

Security reports are especially valuable when they affect:
- runtime safety boundaries
- data handling and storage exposure
- sandbox isolation and execution controls
- LLM prompt integrity and model-output trust boundaries

## Response Expectations

This is a best-effort open source project. Timelines are not guaranteed.

Maintainers aim to:
- acknowledge reports when possible
- validate impact
- patch or document mitigations when feasible
- credit reporters where appropriate

## In Scope

- Code in this repository
- Default runtime/config behavior

## Out of Scope

- Third-party hosted services and model providers
- Local misconfiguration in private deployments
